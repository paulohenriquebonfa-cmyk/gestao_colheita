# Checklist de Backup, Restore e Retencao

## Objetivo

Proteger dados reais, permitir testes com seguranca e evitar perda acidental de historico.

## Checklist

- Confirmar tipo da operacao.
- Confirmar se ha pendencias de sincronizacao.
- Confirmar onde os dados existem: aparelho, nuvem ou ambos.
- Fazer backup antes de acao destrutiva.
- Executar restore ou limpeza conforme escopo.
- Revalidar dados apos a operacao.
- Confirmar se a nuvem tambem foi afetada.
- Registrar impacto final e risco residual.
